# README — AI 診斷日誌 (AI Diagnostic Log)

本專案在開發過程中，透過與 AI 的互動進行問題診斷與程式優化。以下紀錄在開發過程中遇到的重要問題、調整策略與反思。

---

## 1. Buffer 計算流程優化

在建立河川風險區的 buffer 時，最初的設計流程為：

1. 先對河川資料進行 `dissolve`
2. 再依序建立三層 buffer（高 / 中 / 低風險）

然而，此流程在實際執行時運算效率較低，處理時間明顯過長。  
經重新檢視流程並與 AI 討論後，調整為：

1. **先對原始河川資料建立 buffer**
2. **再對 buffer 結果進行 dissolve**

此方式大幅減少幾何運算負擔，使整體處理速度顯著提升。

## 2. 行政區彙總時的唯一值問題

在進行鄉鎮市區層級的統計分析時，需要彙總以下資訊：

- 各行政區 **高 / 中 / 低風險避難所數量**
- 各行政區 **風險區內避難所總收容量**
- 各行政區 **安全避難所總收容量**

最初的作法僅使用 `TOWNNAME` 作為行政區識別欄位，但在實際資料中發現：

> **不同縣市可能存在相同的 TOWNNAME**

因此僅使用 `TOWNNAME` 會導致行政區資料錯誤合併。

在檢查資料屬性後，改為使用：COUNTYNAME + TOWNNAME

作為唯一鍵值，以確保行政區統計結果的正確性。

## 3. 風險人口估算方法的修正

作業指引中提到：

> 「安全區收容量 < 風險區內常住人口的一定比例」

然而此定義較為模糊，未明確說明如何計算「風險區常住人口」。

經與 AI 討論後，新增資料來源：
114年6月行政區人口統計_鄉鎮市區.csv

並建立以下估算流程：

1. 計算 **每個鄉鎮市區的風險區面積比例**
2. 以該比例推估 **風險區常住人口**
3. 以 **風險區常住人口 × 0.2（20%）** 作為分析門檻

此方法能更合理地反映各行政區的實際人口分布情況，使避難容量評估更具依據。

## 結論與反思

透過本次開發經驗可以發現：

- AI 在資料分析過程中，**可能無法完全理解資料本身的結構與屬性**，因此有時會提供不完全正確或不夠嚴謹的建議。
- 在使用 AI 協助分析時，**仍需對每個步驟進行合理性檢查與邏輯驗證**。
- 當研究者遇到難以解決的問題時，AI 仍然可以提供多種可能的解法，**作為後續分析與決策的重要參考**。

因此，在實務應用中，AI 更適合作為 **輔助分析工具**，而非完全取代研究者的判斷。
